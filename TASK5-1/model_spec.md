## 2. 问题定义

- **类型**：二分类
- **目标变量 Y**：
  - `True` → 财报公布次日股价上涨
  - `False` → 财报公布次日股价下跌 / 持平
- **类别分布**：False 59.6% / True 40.4%（轻度不平衡）
- **基线准确率**：59.6%（始终预测多数类）

## 4. Train / Test 划分策略

**采用方案 A：按日期划分。**

| 集合 | 时间范围 |
|------|---------|
| 训练集 | 2021-06-30 ~ 2022-03-31 |
| 测试集 | 2022-04-01 ~ 2022-06-30 |

- ✅ 完全杜绝时间泄漏，最严格的时间序列划分方式
- ✅ 训练和测试覆盖不同股票在不同时间的截面，符合真实预测场景
- ⚠️ 测试集仅覆盖一个季度，市场风格可能与训练集存在差异

> 训练集内可进一步使用时间序列交叉验证（`TimeSeriesSplit`）来调参，避免在单一验证集上过拟合。

## 5. 特征工程

### Step 1：极值处理（Winsorize）

- **方法**：1% / 99% 分位数截尾
- **对象**：全部数值特征
- **原因**：PE、PCF、PS 等估值指标存在极端值（max 可达数十万），会严重干扰模型

### Step 2：共线性诊断与处理

- **方法**：
  1. 计算特征间 Pearson 相关系数矩阵，可视化热力图
  2. 计算各特征的 VIF（Variance Inflation Factor）
- **阈值**：VIF > 10 为严重共线性，VIF > 5 需关注
- **预期高共线性特征对**：

| 预期共线特征对 | 原因 |
|------|------|
| 市盈率PE(TTM) ↔ 市盈率PE(TTM,扣除非经常性损益) | 概念高度重叠，仅相差非经常性损益 |
| 净利润同比增长率 ↔ 利润总额(同比增长率) | 利润总额是净利润的前置环节 |
| 净利润同比增长率 ↔ 基本每股收益(同比增长率) | EPS = 净利润 / 总股本，高度相关 |
| 营业利润(同比增长率) ↔ 利润总额(同比增长率) | 营业利润是利润总额的主要构成 |
| 市现率PCF(现金净流量TTM) ↔ 市现率PCF(经营现金流TTM) | 同一概念的两种口径 |

- **处理策略**：
  1. 经 VIF 诊断后，对 VIF > 10 的特征对，保留业务含义更直接的一个（如保留 PE(TTM)，去掉扣非 PE）
  2. 产生两套特征集用于对照：
     - **特征集 A（去共线性）**：剔除高共线性特征后保留的低 VIF 特征集
     - **特征集 B（全量）**：Winsorize 后不经共线性处理的全量 17 个特征
  3. 逻辑回归、MLP 等线性模型**必须使用特征集 A**，避免共线性导致系数不稳定
  4. 树模型（RF、XGBoost、LightGBM）**同时使用两套特征集**，对照共线性剔除是否影响树模型表现

### Step 3：标准化（部分模型需要）

- **方法**：StandardScaler（Z-score）或 RobustScaler
- **对象**：全部数值特征
- **适用范围**：逻辑回归、MLP 等线性/神经网络模型
- **不需要的模型**：随机森林、XGBoost、LightGBM

### Step 4：特征交叉

**不采用。** PE × 净利润增速、PB × 净资产增速等交叉项缺乏清晰的业务含义，且会引入额外的共线性问题，不纳入特征工程。

## 6. 候选模型

### 6.0 基准模型

始终预测 `False`，准确率 59.6%，AUC = 0.50。

### 6.1 逻辑回归 (`lr`)

| 项目 | 说明 |
|------|------|
| 类型 | 线性模型 |
| 优先级 | ⭐⭐⭐ 高 |
| 优点 | 可解释性强，系数即边际效应，训练快 |
| 缺点 | 无法捕捉非线性关系，需要特征标准化 |
| 需调参 | `C`: [0.01, 0.1, 1, 10]; `penalty`: L1/L2; `class_weight`: balanced/null |

### 6.2 随机森林 (`rf`)

| 项目 | 说明 |
|------|------|
| 类型 | 集成树模型 |
| 优先级 | ⭐⭐⭐ 高 |
| 优点 | 无需标准化，鲁棒性强，可输出特征重要性 |
| 缺点 | 无法外推 |
| 需调参 | `n_estimators`: [100, 200, 500]; `max_depth`: [5, 10, 15, null]; `min_samples_split`: [2, 5, 10] |

### 6.3 XGBoost (`xgb`)

| 项目 | 说明 |
|------|------|
| 类型 | 梯度提升树 |
| 优先级 | ⭐⭐⭐ 高 |
| 优点 | 竞赛级性能，支持早停和正则化，通常表现最佳 |
| 缺点 | 参数多，调优复杂 |
| 需调参 | `n_estimators`: [100, 200, 500]; `max_depth`: [3, 5, 7]; `learning_rate`: [0.01, 0.05, 0.1]; `subsample`: [0.7, 0.8, 1.0]; `scale_pos_weight` |

使用早停，`early_stopping_rounds=20`。

### 6.4 LightGBM (`lgbm`)

| 项目 | 说明 |
|------|------|
| 类型 | 梯度提升树 |
| 优先级 | ⭐⭐ 中 |
| 优点 | 比 XGBoost 更快，内存更小 |
| 缺点 | 小数据集可能过拟合 |
| 需调参 | `n_estimators`: [100, 200, 500]; `max_depth`: [3, 5, 7]; `learning_rate`: [0.01, 0.05, 0.1]; `num_leaves`: [15, 31, 63] |

### 6.5 多层感知机 (`mlp`)

| 项目 | 说明 |
|------|------|
| 类型 | 神经网络 |
| 优先级 | ⭐ 低 |
| 优点 | 可捕捉复杂非线性关系 |
| 缺点 | 数据量可能不够（2万条），可解释性差 |
| 需调参 | `hidden_layer_sizes`: [[64,32], [128,64]]; `activation`: relu/tanh; `alpha`: [1e-4, 1e-3] |

> **推荐组合**：LR + XGBoost — LR 负责解释，XGB 负责精度。

## 7. 评估方案

### 7.1 核心指标

| 指标 | 说明 |
|------|------|
| **AUC-ROC** | 主要指标，不受阈值影响，评估排序能力 |
| Accuracy | 整体准确率 |
| Precision | 预测"涨"中有多少真的涨 |
| Recall | 实际"涨"中被识别出来的比例 |
| F1-Score | Precision 和 Recall 的调和平均 |

### 7.2 可解释性

| 方法 | 说明 |
|------|------|
| 特征重要性 | 树模型输出各特征对预测的贡献排名 |
| SHAP 值 | 解释每个预测的具体驱动因素 |
| 逻辑回归系数 | 系数符号 + 显著性（p 值），边际效应解释 |

## 8. 稳健性检验

### 8.1 纯成长指标建模

- 仅用 8 个增长率变量建模，去掉估值类特征
- 目的：评估估值指标是否提供增量信息

### 8.2 共线性处理对照（树模型）

- 树模型分别使用特征集 A（去共线性）和特征集 B（全量）建模，对比 AUC
- 目的：验证共线性剔除对树模型是否有负面影响

### 8.3 不同市场环境子集评估

- 按测试集涨跌家数比例分组评估
- 目的：确认模型在牛/熊市表现是否一致
