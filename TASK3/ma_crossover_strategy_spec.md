# 双均线交叉趋势跟踪策略 — 设计规范

> **策略 ID**: MA_CROSSOVER_V1 &emsp; **版本**: 1.0.0 &emsp; **日期**: 2026-07-07  
> **分类**: 趋势跟踪 &emsp; **适用市场**: A 股（单向做多）

---

## 一、策略概述

基于两条不同周期移动平均线的相对位置产生买卖信号。快线上穿慢线（金叉）做多，快线下穿慢线（死叉）平仓。这是一个经典的**趋势跟踪策略**——在单边市中表现优异，在震荡市中容易产生假信号。

### 1.1 为什么均线交叉能赚钱？

三个核心逻辑，层层递进：

```
价格 = 趋势 + 噪声
         ↓
   均线取平均值 → 消除随机噪声 → 暴露出底层趋势
         ↓
   快线穿越慢线 → 短期共识偏离中期成本 → 趋势方向可能改变
         ↓
   确认趋势后跟随 → 捕捉趋势形成后的惯性利润
```

**它赚的是什么钱？** 市场的动量效应——上涨的标的倾向于继续上涨，下跌的标的倾向于继续下跌。均线交叉策略捕捉的是趋势形成后、惯性延续的这段利润。

**关键认知：** 均线交叉不是预测工具，而是确认工具。它不回答"接下来会怎样"，它回答"过去 N 天的平均价格已经发生了什么变化"。滞后性是它的成本，也是它避免假突破的过滤机制。

## 二、参数定义

### 2.1 均线类型：SMA vs EMA

| 特性 | SMA（简单移动平均） | EMA（指数移动平均） |
|------|---------------------|---------------------|
| 权重分配 | 过去 n 天等权重（各 1/n） | 近期权重更大，指数衰减 |
| 公式 | mean(Pₜ, Pₜ₋₁, ..., Pₜ₋ₙ₊₁) | α·Pₜ + (1-α)·EMAₜ₋₁, α = 2/(n+1) |
| 响应速度 | 较慢，有窗口效应 | 较快，无窗口效应 |
| 对旧数据的处理 | 第 n+1 天旧数据被踢出，可能产生毛刺 | 旧数据权重趋近于 0 但永不为 0 |
| 推荐场景 | 需要严格平滑、不在乎延迟的长线分析 | 需要及时响应趋势变化的交易策略 |

> **默认选择：EMA。** 实操中 EMA 更常用，因为：(1) 对趋势转向更敏感，(2) 不会因旧数据掉出窗口而产生毛刺，(3) 递归计算，数学性质更好。

### 2.2 周期参数

| 参数 | 默认值 | 范围 | 含义 |
|------|--------|------|------|
| 快线周期（fast_period） | 10 | 3 ~ 30 | 反映近期市场共识，决定信号的灵敏度 |
| 慢线周期（slow_period） | 60 | 20 ~ 250 | 反映中期持仓成本，决定信号的可靠性 |

**预设档位：**

| 风格 | 快线 | 慢线 | 适用场景 |
|------|------|------|---------|
| 激进 | 5 | 20 | 高频短线，信号密集，胜率低但盈亏比高 |
| 均衡（推荐） | 10 | 60 | 月度级别趋势，平衡信号质量与频率 |
| 保守 | 20 | 120 | 半年级别大趋势，信号稀少但可靠性高 |

### 2.3 参数约束

1. **快线必须严格小于慢线：** `fast_period < slow_period`，否则计算无意义。
2. **避免整数倍关系：** 快线和慢线不应有公约数。如避免 10/30（30 是 10 的倍数），推荐 10/27。这可以防止信号过度受某个特定时间段数据的影响。
3. **周期比在合理范围内：** `slow_period / fast_period` 通常控制在 3 ~ 12 之间。比值太小（如 5/8）两条线几乎贴合，信号太密；比值太大（如 3/200）慢线过于迟钝，信号太稀。

## 三、信号逻辑

### 3.1 金叉买入

```
条件: MA_fast(t-1) ≤ MA_slow(t-1) 且 MA_fast(t) > MA_slow(t)
动作: 买入（开多）
```

快线从下方向上穿越慢线，代表短期价格动能超过中期趋势，多头趋势启动。

> **风险：假金叉。** 在横盘震荡中频繁出现——价格短暂冲高后回落，买在局部高点。

### 3.2 死叉卖出

```
条件: MA_fast(t-1) ≥ MA_slow(t-1) 且 MA_fast(t) < MA_slow(t)
动作: 卖出（平多）
```

快线从上方向下穿越慢线，代表短期价格动能跌破中期成本，空头趋势启动。

> **风险：假死叉。** 趋势中的正常回调导致短暂下穿，价格随后恢复上涨，你却在回调底部卖出了。

### 3.3 仓位状态机

```
        金叉                 死叉
空仓(0) ────→ 满仓(1) ────→ 空仓(0)
  ↑                                    │
  └────────────────────────────────────┘
              (死叉后回到空仓)
```

只有两种状态：**全仓** 或 **空仓**。没有分批建仓，没有加减仓，没有做空。

### 3.4 执行时序：T+1 规则

这是一个很容易出错但至关重要的细节：

1. **t 日收盘后**：计算当日收盘价下的均线值，判断是否产生交叉信号。
2. **t+1 日开盘**：以开盘价执行交易。

为什么要这样做？如果在 t 日收盘前就执行交易，等于你用了当天还没确定的收盘价来做决策——这就是**未来函数**（Look-ahead Bias），回测时最常见的自欺欺人。

## 四、回测配置

### 4.1 初始条件

- **初始资金：** 100 万元人民币
- **仓位方式：** 满仓进出（有信号全仓，无信号空仓）
- **基准策略：** Buy & Hold（从起始日全仓买入并持有至结束日）

### 4.2 交易成本

交易成本是"理论收益"到"实际收益"的三大税。忽略其中任何一项，你的回测都是在骗自己。

| 成本项 | 费率 | 收取时点 | 说明 |
|--------|------|---------|------|
| 佣金 | 0.03%（万三） | 买入 + 卖出 | A 股券商标准费率 |
| 印花税 | 0.05%（万五） | 仅卖出 | A 股现行标准，不可协商 |
| 滑点 | 0.1%（千一） | 买入 + 卖出 | 模拟市场冲击成本，比佣金更隐形但更致命 |

> **单次完整交易的总摩擦成本：** 买入约 0.13%，卖出约 0.18%，合计约 0.31%。假设年化换手率 200%（约 4 次完整买卖），年化成本约 **1.2% ~ 1.5%**。这意味着策略 Alpha 必须超过这个数才算真正赚钱。

### 4.3 数据要求

| 项目 | 要求 |
|------|------|
| 频率 | 日线 |
| 必需字段 | trade_date, open, high, low, close |
| 推荐复权 | 前复权（QFQ），确保价格连续无除权跳空 |
| 最少历史数据 | max(fast_period, slow_period) + 100 根 K 线 |

均线需要预热期才能计算——比如 120 日均线需要前 119 天的数据才能算出第一个值。额外 100 根 K 线作为缓冲。

## 五、风控规则

这些规则不是"可以有的"，是"必须有的"。没有风控的策略不是策略，是赌博。

### 5.1 最大回撤熔断

> 净值从历史最高点回撤 **≥ 25%** → 暂停交易，重新评估策略有效性。

不要幻想"回撤只是暂时的"。25% 的回撤需要 33% 的涨幅才能回本，50% 的回撤需要 100% 的涨幅。

### 5.2 连续亏损暂停

> 连续亏损 **5 次** 后 → 暂停 10 个交易日。

连续亏损通常是市场环境变化的信号（从趋势市切换到了震荡市），继续用同样的参数去硬扛只会增加无谓损失。

### 5.3 极端波动日保护

> 当日涨跌幅超过 **±10%**（涨跌停板）→ 不执行新开仓信号（已有仓位正常平仓）。

涨停板买不到，跌停板卖不掉。在这类日子触发的信号没有执行条件，不应计入回测统计。

### 5.4 单票上限（多票扩展时）

本策略默认单票满仓。如扩展为多票组合，建议单票上限不超过 **25%**。

## 六、预期表现：不同市况下的行为

均线交叉策略的收益来源不对称——它在趋势市中赚到的钱需要覆盖震荡市中的反复亏损。

| 市况 | 相对基准表现 | 典型年化夏普 | 典型胜率 | 原因 |
|------|-------------|-------------|---------|------|
| 单边上涨（牛市） | **强于基准** | 1.0 ~ 2.5 | 35% ~ 50% | 金叉后持续持有多头，捕捉完整波段 |
| 单边下跌（熊市） | 持平或略优 | 0.5 ~ 1.5 | — | 死叉后空仓躲过大跌，但死叉滞后已回吐利润 |
| 横盘震荡（猴市） | **弱于基准** | -1.0 ~ 0.5 | 20% ~ 35% | 鞭梢效应：反复穿越均线产生连续小额亏损 |

### 鞭梢效应（Whipsaw）

这是均线交叉策略在震荡市中亏钱的根本原因：

```
震荡市中的典型假信号序列：
  买→小跌→止损→反弹→再买→小跌→止损→反弹→再买...
  
每次亏损 1%~3%，10 次假信号就能吞噬 10%~30% 的本金。
而趋势跟踪策略的优势——一波大趋势赚 30%~50%——在震荡市中完全没有发挥机会。
```

**缓解措施：** 增加趋势过滤层（如 ADX > 25 时允许交易，否则空仓），可以在震荡市中大幅减少交易次数，但代价是趋势启动初期也会延迟入场。

## 七、常见变体

### 7.1 三均线系统

增加一条中线，形成快/中/慢三层过滤：

```
MA5 上穿 MA20  →  预警信号（趋势可能启动）
MA20 上穿 MA60 →  确认信号（正式入场）
```

减少假信号，但确认过程更长，入场更晚。

### 7.2 均线交叉 + 趋势过滤

最简单也最有效的改进：金叉信号需要同时满足 `price > MA200`。

这样你在 MA200 下方的所有金叉都不会入场——这些往往是熊市中的反弹假信号。代价是熊转牛的第一段涨幅会被过滤掉。

### 7.3 均线交叉 + 成交量确认

金叉日成交量需超过 20 日均量的 **1.5 倍**。

逻辑很简单：无量假突破。真正的趋势启动一定有增量资金推动，如果金叉当天的成交量还不如平时，那大概率是假的。

### 7.4 海龟通道突破（均线思想的极端形式）

用 Donchian 通道代替均线：

- 快线 = 过去 20 日最高价
- 慢线 = 过去 20 日最低价

本质上是均线交叉的思想扩展到价格极值。突破 20 日最高价做多，跌破 20 日最低价做空。海龟交易法还配套了 ATR 计算的加仓和止损规则。

## 八、验证清单

实现策略后，逐一对照以下检查项。任何一项未通过，回测结果都不应该被信任。

### 8.1 数据完整性

- [ ] 价格数据无缺失行（缺失日需填充前收或标记为停牌）
- [ ] 复权已正确应用（前复权价格无跳空，除权除息后价格连续性已确认）
- [ ] 停牌日的处理规则已明确（跳过 / 延续前一日仓位）
- [ ] 涨跌停日不可交易的处理规则已明确

### 8.2 信号正确性

- [ ] 金叉定义：MA_fast 从 ≤ 变为 > MA_slow（不是简单的 fast > slow）
- [ ] 死叉定义：MA_fast 从 ≥ 变为 < MA_slow
- [ ] 金叉和死叉互斥（同一根 K 线不可能同时产生两种信号）
- [ ] 首根 K 线不产生信号（均线需要预热期）
- [ ] **信号确认日 ≠ 实际交易日**（确认在 t 日收盘，交易在 t+1 日开盘）

### 8.3 成本核算

- [ ] 佣金在买入和卖出时各收取一次（双边）
- [ ] 印花税仅在卖出时收取（A 股规则）
- [ ] 滑点在每次交易时按成交金额比例扣除
- [ ] 现金余额不能为负（仓位 × 价格不能超过可用资金）

### 8.4 指标计算

| 指标 | 公式 |
|------|------|
| 总收益率 | (期末净值 / 期初净值) - 1 |
| 年化收益率 | (1 + 总收益)^(252 / 交易日数) - 1 |
| 年化夏普比率 | √252 × (日超额收益均值 / 日超额收益标准差) |
| 最大回撤 | min(每日净值 / 累计最高净值 - 1) |
| Calmar 比率 | 年化收益率 / | 最大回撤 | |
| 胜率 | 盈利交易笔数 / 总交易笔数 |
| 盈亏比 | 平均盈利金额 / 平均亏损金额（取绝对值） |

## 九、参数敏感性分析

**目的：** 验证策略是否对特定参数值过度依赖。如果小幅调整参数导致收益率剧烈变化，说明策略可能过拟合了某一段特定数据。

**测试网格：**

| 维度 | 测试值 |
|------|--------|
| 快线周期 | 5, 10, 15, 20 |
| 慢线周期 | 30, 60, 90, 120 |
| 均线类型 | EMA, SMA |

组合总数：4 × 4 × 2 = **32 组**

**通过标准：** 在至少 60% 的参数组合中，策略年化夏普 > 0。这说明策略的逻辑在不同参数下都有一致性，不是"碰巧调出来的一组好参数"。

**警戒信号：** 最优参数组合 ±20% 的变动导致夏普下降 **> 50%** → 标记为"参数敏感"，策略可能过拟合，需人工审查。

## 十、输出规范

### 10.1 回测报告（HTML）

三个必需图表 + 九个绩效指标：

| 图表 | 包含内容 |
|------|---------|
| 价格 & 均线 & 信号 | 收盘价曲线 + 快线 + 慢线 + 金叉（绿三角）+ 死叉（红倒三角） |
| 资金曲线 | 策略净值 vs 基准净值（Buy & Hold）的双线对比 |
| 回撤曲线（可选） | 策略回撤百分比 vs 基准回撤百分比 |

### 10.2 绩效指标

策略总收益率、基准总收益率、年化收益率、年化夏普比率、最大回撤、Calmar 比率、胜率、盈亏比、总交易次数。

### 10.3 CSV 输出

- **逐日信号日志：** trade_date, price, ma_fast, ma_slow, signal, cross, position, strategy_return, equity
- **逐笔交易记录：** entry_date, entry_price, exit_date, exit_price, return, holding_days

## 十一、局限性

### 固有缺陷（无法消除，只能接受）

- **滞后性：** 信号永远在趋势已经启动后才出现。这是趋势确认的必要成本。任何试图"提前"的做法都会增加假信号。
- **鞭梢效应：** 震荡市中的连续小亏是趋势跟踪策略的结构性亏损，无法彻底消除，只能靠趋势过滤减少。
- **趋势持续性未知：** 你不知道当前趋势会持续多久——可能在趋势刚好结束时买入。
- **参数敏感：** 不同周期组合在同一个品种上表现差异巨大，没有一组"万能参数"。

### 实际限制

- 滑点和冲击成本在极端行情中可能被严重低估。
- 跌停板时卖出信号无法执行，需特殊处理。
- T+1 制度下隔夜跳空风险无法规避。
- 大资金运作需额外考虑市场深度和流动性。

### 适用范围

本 spec 仅针对 **A 股单向做多**场景，不涉及多品种组合、对冲、期权、融资融券、跨境市场等。

> **重要声明：** 回测结果不代表实盘表现。过去的数据是确定的，未来的市场是不确定的。任何策略在上实盘之前，至少需要经过样本外验证和模拟盘观察。

## 十二、扩展路线图

本 spec 为最小可行版本（MVP），以下是按优先级排序的扩展方向：

| 优先级 | 扩展方向 | 说明 |
|--------|---------|------|
| **高** | 趋势过滤层 | 加入 ADX 或长期均线作为趋势确认，减少震荡市假信号 |
| **高** | 动态仓位管理 | 用 ATR 或凯利公式替代满仓进出，实现风险约束下的仓位优化 |
| 中 | 多品种回测 | 支持 5+ 个品种同时回测，输出横向对比报告 |
| 中 | 参数自动优化 | 网格搜索 + 样本外验证，自动寻找最优快慢周期组合 |
| 低 | 港股/美股适配 | 扩展支持港股（T+0，无涨跌停）和美股（T+2 结算）的交易规则 |
