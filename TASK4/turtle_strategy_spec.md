# 海龟策略回测 Spec

> **版本**: 1.0.0 | **创建日期**: 2026-07-11 | **输出目录**: `TASK4/output/`

基于已存储的三只 A 股日线数据，实现完整海龟趋势跟踪策略（双系统 + 金字塔加仓 + 波动率仓位管理 + 多层风控），生成交易信号、持仓记录和回测绩效报告。**本文件为纯规范，不包含代码实现。**

---

## 二、策略核心参数

> 以下参数均可自由调整。修改后重新运行回测即可对比效果。每个参数都标注了经典值和建议范围。

### 2.1 趋势跟踪 — 系统1（短线）

| 参数 | 值 | 说明 |
|------|-----|------|
| `entry_period` | **20** | 入场：价格突破过去 20 日最高价 → 买入信号 |
| `exit_period` | **10** | 出场：价格跌破过去 10 日最低价 → 卖出信号 |
| `reentry_filter` | **true** | 如果上一次系统1信号最终以亏损出场，跳过当前入场信号 |
| 典型胜率 | 35%~40% | 信号频率高、胜率偏低、盈亏比大 |

> ⚠ 减小 `entry_period` → 更敏感、信号更多、假突破更多；增大 → 更迟钝、信号更少。

### 2.2 趋势跟踪 — 系统2（长线）

| 参数 | 值 | 说明 |
|------|-----|------|
| `entry_period` | **55** | 入场：价格突破过去 55 日最高价 → 买入信号 |
| `exit_period` | **20** | 出场：价格跌破过去 20 日最低价 → 卖出信号 |
| `skip_signal` | **false** | 系统2信号**永不跳过**，每次必须执行 |
| 典型胜率 | 40%~45% | 信号稀疏但更可靠，捕捉大级别趋势 |

### 2.3 入场订单类型

| 参数 | 值 |
|------|-----|
| 类型 | Buy Stop（买入停止单） |
| 触发规则 | `close_qfq` 突破唐奇安通道上轨时触发 |
| 回测模拟 | 以**次日开盘价** `open_qfq` 成交（模拟次日挂单执行） |
| 滑点 | `0.1%`（`slippage_pct = 0.001`） |

### 2.4 ATR — 平均真实波幅

| 参数 | 值 |
|------|-----|
| 周期 | **20** |
| 方法 | 指数平滑（Wilder 方法） |

**计算步骤：**

1. **True Range**：`TR[t] = max(high[t] - low[t], |high[t] - close[t-1]|, |low[t] - close[t-1]|)`
2. **初始 ATR**（第 20 天）：`ATR = avg(TR[0:20])`
3. **递推**（第 21 天起）：`ATR[t] = (19 × ATR[t-1] + TR[t]) / 20`

### 2.5 Unit — 交易单位

```
1 Unit = floor( Account × 0.01 / (N × 100) ) × 100
```

| 参数 | 值 | 说明 |
|------|-----|------|
| `risk_per_trade_pct` | **0.01** | 单笔风险占账户 1% |
| `lot_size` | **100** | A 股每手 100 股，Unit 向下取整到 100 的倍数 |
| `account_initial` | **1,000,000** | 初始资金 100 万人民币 |
| N | = ATR(20) | 平均每日波动幅度 |

> 含义：如果这笔交易亏了 1 个 N，账户恰好损失 1%。修改 `risk_per_trade_pct` 可调整激进程度：`0.005` = 保守，`0.02` = 激进。

### 2.6 金字塔加仓

| 参数 | 值 |
|------|-----|
| `max_units` | **4** |
| `add_interval_n` | **0.5** |

**规则**：首次建仓 1 Unit。价格每向有利方向运行 0.5 N，加仓 1 Unit，最多 4 Unit。

```
第 k 次加仓价 = 首次入场价 + (k-1) × 0.5 × N
```

### 2.7 初始止损

| 参数 | 值 |
|------|-----|
| `stop_distance_n` | **2.0** |

**规则**：`止损价 = 入场价 - 2.0 × N`。建仓后如果价格反向运行 2.0 N，立即平仓。

> ⚠ 止损距离 × Unit 风险 = 实际最大亏损（2.0 × 1% = 2%）。止损使用入场时的 N 值锚定，不随每日 ATR 更新。

### 2.8 移动出场（跟踪止盈）

不设固定止盈价。出场信号仅由唐奇安通道下轨触发（跌破 M 日最低价）：

| 系统 | 出场周期 M |
|------|-----------|
| 系统1 | **10** 日 |
| 系统2 | **20** 日 |

### 2.9 市场相关性限制 — 三层风控

即使每个品种分开看都没超限，加在一起也可能超限。三层嵌套，逐层检查：

#### 第一层：单一股票 ≤ 4 Unit

任何单只股票的持仓不超过 4 Unit。

#### 第二层：关联组 ≤ 6 Unit

| 组名 | 成员 | 上限 | 说明 |
|------|------|------|------|
| 半导体组 | `688981.SH`（中芯国际）、`603986.SH`（兆易创新） | 6 Unit | 同受半导体周期驱动 |
| 新能源/汽车组 | `002594.SZ`（比亚迪） | 6 Unit | 独立板块 |

> 例：中芯国际 4U + 兆易创新 2U = 6U ✓；中芯国际 4U + 兆易创新 3U = 7U ✗（超限）

#### 第三层：同一方向 ≤ 12 Unit

所有做多仓位合计不超过 12 Unit。三只股票 × 4 Unit = 12，恰好等于上限，但半导体组内限制 6 会让这个上限在实际中更早触发。

### 2.10 全局回撤熔断（可选）

| 参数 | 值 | 说明 |
|------|-----|------|
| `enabled` | **false** | 海龟原版无此规则，默认关闭 |
| `max_drawdown_pct` | 0.30 | 账户回撤超 30% 时暂停新开仓 |

### 2.11 A+H 双市场（默认关闭）

| 参数 | 值 |
|------|-----|
| `enable_hk` | **false** |

开启后，中芯国际和比亚迪各有 A 股 + 港股两个独立策略实例。同一标的的 A+H 合计 ≤ 4 Unit。港股交易日历与 A 股不同，需独立日期对齐。

## 三、信号生成流程（8 步 pipeline）

对每只启用的股票逐日计算。所有步骤仅依赖当天及之前的数据，严格避免未来函数。

> **运行模式说明**：跨股票对比需要两种模式都跑——
> - **独立模式**：每只股票各自独立账户，无跨股票风控干扰。用于回答"海龟对哪只股票有效"（生成 4.5 对比图）。
> - **组合模式**：三只股票共享一个账户，受三层相关性限制约束。用于回答"海龟组合实战效果如何"（生成 4.3/4.4）。
>
> 两种模式共用 Step 1~6，区别仅在于 Step 7（风控检查）。

### Step 1 — 加载与准备

加载 `daily_adjusted.csv`，按 `trade_date` 升序排列。

**输入**: `price_file`  
**输出**: `df[trade_date, open_qfq, high_qfq, low_qfq, close_qfq, vol, amount]`

### Step 2 — 计算 True Range

```
TR[t] = max( high[t] - low[t], |high[t] - close[t-1]|, |low[t] - close[t-1]| )
```

第一天（无昨日收盘）TR = high - low。

**输出**: `df` 新增 `tr` 列

### Step 3 — 计算 ATR(20)

```
初始:    ATR[20] = avg(TR[0:20])
递推:    ATR[t]  = (19 × ATR[t-1] + TR[t]) / 20
```

**输出**: `df` 新增 `n` 列（即 ATR(20)）

### Step 4 — 计算唐奇安通道

两种周期并行，不含当日数据：

| 通道 | 公式 |
|------|------|
| 系统1 上轨 | `max(high[t-20 : t-1])` |
| 系统1 下轨 | `min(low[t-10 : t-1])` |
| 系统2 上轨 | `max(high[t-55 : t-1])` |
| 系统2 下轨 | `min(low[t-20 : t-1])` |

**输出**: `df` 新增 `sys1_upper, sys1_lower, sys2_upper, sys2_lower` 列

### Step 5 — 生成信号

| 信号类型 | 条件 | 动作 |
|----------|------|------|
| **S1_BUY** | `close[t] > sys1_upper[t]` 且无系统1持仓 | 生成买入信号，入场价 = `open[t+1] × 1.001` |
| | 过滤规则 | 上一次系统1信号亏损出场 → 跳过本次 |
| **S1_SELL** | `close[t] < sys1_lower[t]` 且有系统1持仓 | 生成卖出信号，出场价 = `open[t+1] × 0.999` |
| **S2_BUY** | `close[t] > sys2_upper[t]` 且无系统2持仓 | 生成买入信号，不跳过 |
| **S2_SELL** | `close[t] < sys2_lower[t]` 且有系统2持仓 | 生成卖出信号 |

### Step 6 — 仓位计算

入场信号触发时，基于当日最新 N 值和账户净值：

```
1 Unit = floor( account_equity × 0.01 / (N × 100) ) × 100
```

入场后每 0.5 N 加仓，最多 4 Unit，每次以次日开盘价成交。

### Step 7 — 风控检查

每个信号开仓前，按顺序检查三层上限。超出上限的信号标记为 **REJECTED** 并记录原因：

1. 该股票当前 Unit 数 + 新增 ≤ 4
2. 关联组总 Unit 数 + 新增 ≤ 6
3. 所有做多持仓总 Unit 数 + 新增 ≤ 12

### Step 8 — 止损检查

每日对所有持仓：

```
如果 low[t] < (入场价 - 2.0 × N_入场时) → 以止损价平仓
```

> 止损使用入场时的 N 值（锚定不变），避免因波动加大而被动放宽止损。

### 4.5 跨股票策略对比 — `turtle_comparison.html`

**核心目标**：回答"海龟策略对哪只股票 / 哪类股票更有效"。

这是一个独立的 HTML Dashboard，包含以下对比图表（使用 ECharts 渲染）：

#### 面板 1：绩效雷达图（六维度）

| 维度 | 说明 | 怎么看 |
|------|------|--------|
| 年化收益率 | 越高越好 | 趋势跟踪在强趋势股上得分高 |
| 最大回撤 | 越低越好（取反向） | 高波动股回撤大，趋势股回撤可控 |
| 夏普比率 | 越高越好 | 综合衡量风险调整收益 |
| 胜率 | 越高越好 | 趋势清晰的股票胜率偏高 |
| 盈亏因子 | > 1.5 为优秀 | 趋势股的盈亏比通常碾压震荡股 |
| 平均持仓天数 | 越长 = 趋势越持续 | 短的说明假突破多 |

六根轴叠加三只股票的三条多边形，直观对比"谁更适合趋势跟踪"。

#### 面板 2：净值曲线对比

三条曲线叠加在同一坐标系中，每条曲线 = 该股票**独立跑海龟策略**的账户净值变化（初始资金相同）：

- X 轴：时间
- Y 轴：账户净值（从初始资金标准化起步）
- 颜色区分三只股票 + 一条买入持有基准线

#### 面板 3：月度收益热力图

| X 轴 | Y 轴 | 颜色 |
|------|------|------|
| 月份（1~12月） | 年份 | 红 = 盈利月，绿 = 亏损月，深浅 = 幅度 |

用于发现策略的季节性特征或特定年份的系统性失效。

#### 面板 4：特征对比表

一张汇总表格，在回测结果外附加每只股票的**基本特征**，帮助判断"什么类型的股票适合海龟"：

| 对比维度 | 来源 |
|----------|------|
| 区间总收益率 / 年化 / 夏普 / 最大回撤 | 回测结果 |
| 区间日均波动率（区间 N 均值） | `daily_adjusted.csv` |
| 区间趋势强度（上涨日占比、连涨天数均值） | 衍生计算 |
| 区间振幅（最高价 / 最低价 - 1） | `daily_adjusted.csv` |
| 市值区间（从 daily_basic 取 `total_mv`） | `daily_basic.csv` |
| 所属板块 | 手动标注（半导体 / 新能源） |

> 💡 **分析思路**：回测结束后，对比三只股票的特征 × 绩效。如果半导体组（中芯国际、兆易创新）的绩效接近而比亚迪差异大，说明策略效果与**行业趋势的持续性**有关；如果高波动股绩效差、低波动股绩效好，说明策略更适合低噪声的趋势环境。

### 4.6 样本量提醒

当前仅 3 只股票、约 2~3 年数据，**不足以得出"海龟策略适用于某类股票"的统计结论**。跨股票对比的价值在于**提出假设**（如"半导体趋势持续性优于新能源？"），而非验证假设。当股票池扩展到 10+ 只、覆盖更多行业和市值区间后，这些对比才能真正回答策略适用性问题。

## 五、用户修改指南

### 5.1 调整数据区间

无需修改 spec。CSV 覆盖什么日期，回测就跑到什么日期。如需限定区间，在 spec 中增加 `data_start` / `data_end` 字段。

### 5.2 启用 / 停用股票

将对应股票的 `enable` 设为 `true` 或 `false`。例：只测比亚迪 → 中芯国际和兆易创新 `enable: false`。

### 5.3 三档策略风格预设

| 风格 | `risk_per_trade_pct` | `max_units` | `stop_distance_n` | 效果 |
|------|---------------------|-------------|-------------------|------|
| **保守** | 0.005 | 3 | 1.5 | 小仓位、紧止损、低回撤，可能错过部分趋势 |
| **经典** | 0.01 | 4 | 2.0 | Dennis 原版参数，平衡型 |
| **激进** | 0.02 | 5 | 2.5 | 大仓位、宽止损、大波动，需极强心理承受力 |

### 5.4 调整入场周期

- **更快响应**: 系统1 entry=10 exit=5；系统2 entry=40 exit=15 → 更多交易
- **更慢响应**: 系统1 entry=30 exit=15；系统2 entry=70 exit=30 → 更少交易

### 5.5 新增股票

1. 在 `stocks` 中新增一条记录，填写 `ts_code, name, data_dir, price_file`
2. 配置 `price_columns` 字段映射
3. 评估与现有股票的相关性，更新 `correlated_groups`
4. 确保对应 CSV 已存在于 `data/` 目录

### 5.6 启用港股

1. `enable_hk` 设为 `true`
2. 港股继承同一策略参数，但需独立日期对齐
3. A 股和港股同标的视为同一组（合计 ≤ 4 Unit）

## 六、策略评估基准

| 基准 | 说明 |
|------|------|
| 买入持有 (Buy & Hold) | 回测起始日满仓等权重买入，持有至结束。对比总收益、回撤、夏普。 |

## 七、正确性约束（实现时必须遵守）

1. **禁止未来函数**：第 T 日决策仅使用 ≤ T-1 日数据判断，以 T+1 开盘价成交。
2. **止损优先**：每日先检查止损，再检查入场。同日既触发止损又触发入场时，只执行止损。
3. **N 值锚定**：每 Unit 的止损距离使用入场时的 N，不随每日 ATR 更新。加仓时按加仓当日 N 计算，止损锚定在加仓时的 N。
4. **信号独立**：系统1 和系统2 完全独立，各自维护持仓和出场逻辑。同一天可同时持有两套系统的仓位。
5. **整数手数**：A 股每手 100 股，Unit 向下取整到 100 的倍数。
6. **交易成本**：默认不计（简化回测）。如需计入，增加 `commission_rate` 和 `stamp_tax_rate`。
7. **涨跌停处理**：一字涨停/跌停无法成交时信号视为未执行。此功能为高级特性，初版可忽略。
